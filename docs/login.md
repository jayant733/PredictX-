# Frontend Specification: Login Page (`login.md`)

This page handles user authentication via Solana web3 wallets (Phantom, Backpack, Solflare) and integrates with Supabase for session management.

## 1. Visual Design & Theme
- **Theme**: Premium dark-mode (deep space/charcoal background `#0B0C10`, steel blue accent `#1F2833`, neon cyan hover outlines `#66FCF1`, and gold highlights `#IBAS7`).
- **Layout**: Center-aligned login card with a subtle glassmorphism effect (backdrop-blur, thin semi-transparent border, and soft glow shadow).
- **Background**: Animated particle field or subtle grid lines to give a premium crypto/financial trading vibe.

## 2. Key Components

### A. Welcome / Brand Section
- **Logo**: Sleek futuristic icon (Nexus node symbol).
- **Tagline**: "Trade Your Opinions. Shape the Future."
- **Stats Counter (Optional/Nice-to-Have)**: Displays total volume traded or active traders to build trust.

### B. Wallet Login Options Card
- A card featuring a list of supported wallets as high-contrast buttons:
  - **Phantom Wallet Button**: Brand colors (Purple logo, light-purple background on hover).
  - **Backpack Wallet Button**: Brand colors (Red logo, dark red/grey hover).
  - **Generic Solana Wallet Adapter (WalletConnect)**: Clean generic option for other wallets (Solflare, Ledger).
- **Micro-interactions**: Scale button slightly on hover, glowing border transition.

### C. Authentication Flow / Loading States
1. **Wallet Connection**: Clicking a button prompts the browser extension to connect.
2. **Signature Request (Anti-replay/Auth)**:
   - Once connected, show a secondary overlay: "Sign Message to Authenticate".
   - The user must sign a message generated by the backend (e.g., `"Welcome to Nexus! Sign this message to authenticate. Nonce: <random_uuid>"`).
   - Display a spinner: "Waiting for signature...".
3. **Session Verification**:
   - Send the public key and signature to the backend endpoint: `/api/auth/wallet-login`.
   - On success, store the JWT token returned by the server, redirect the user to the `/dashboard` route, and fetch the user profile.

### D. Errors and Fallbacks
- **Wallet Not Installed**: If Phantom or Backpack is not detected, show a clean drawer/modal: "Phantom/Backpack not detected. [Download Phantom](https://phantom.app) | [Download Backpack](https://backpack.app)".
- **User Rejected Signature**: Toast message: "Signature request rejected. Please try again to authenticate."
- **Network Error**: Toast message: "Failed to connect to authentication server. Please check your internet connection."

## 3. Web3 & Supabase Auth API Flow (Technical Detail for Stitch)

```javascript
// Example wallet sign-in flow logic
async function signInWithSolana() {
  // 1. Get Provider (e.g. Phantom or Backpack)
  const provider = window.solana; 
  if (!provider) throw new Error("Wallet not found");

  // 2. Connect to wallet
  const resp = await provider.connect();
  const publicKey = resp.publicKey.toString();

  // 3. Fetch challenge nonce from backend
  const challengeResp = await fetch(`/api/auth/challenge?publicKey=${publicKey}`);
  const { nonce } = await challengeResp.json();

  // 4. Request user signature
  const message = `Welcome to Nexus!\n\nSign this message to authenticate.\nNonce: ${nonce}`;
  const encodedMessage = new TextEncoder().encode(message);
  const signedMessage = await provider.signMessage(encodedMessage, "utf8");

  // 5. Verify signature & get Supabase JWT
  const authResp = await fetch(`/api/auth/verify`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      publicKey,
      signature: Buffer.from(signedMessage.signature).toString("hex"),
      nonce
    })
  });

  const { access_token, user } = await authResp.json();
  
  // 6. Initialize Supabase client session
  supabase.auth.setSession({
    access_token,
    refresh_token: "" // custom setup
  });
}
```
